# ベンチプレストレーナー ロジック & ガイドライン

このスキルは、ベンチプレス管理アプリの開発における計算ロジック、データ処理のルール、およびUI/UXの基準を定義します。

## 1. 計算ロジック

### 推定1RM (Estimated 1RM)
- **計算式**: Epley式を使用します。
  - `1RM = Weight * (1 + Reps / 30)`
- **実装**: `js/data.js` の `estimateMax(weight, reps)` 関数を参照してください。
- **丸め処理**: 小数第1位で四捨五入します。

### 重量の丸め (Weight Rounding)
- **ルール**: 全ての計算された重量は **2.5kg単位** で丸めます。
- **実装**: `js/data.js` の `roundWeight(weight)` 関数を使用してください。

## 2. 自己ベスト (PR) の定義

グラフやPRパネルで表示する「最高記録」には厳格な基準があります。

- **対象種目**: `bench_press`（通常のベンチプレス）のみを対象とします。
- **対象レップ数**:
  - **1RM**: `reps === 1` のセットのみ。
  - **3RM**: `reps === 3` のセットのみ。
  - **5RM**: `reps === 5` のセットのみ。
- **グラフ描画の注意**:
  - 「その日時点での歴代最高記録」が更新された日のみ、点をプロットします。
  - データがない日は `null` を入れ、Chart.js の `spanGaps: true` を設定して線でつなぎます。

## 3. データ構造 (Data Structure)

履歴データ (`history`) は以下のような JSON 形式で `localStorage` または `GAS PropertiesService` に保存されます。

```json
{
  "id": 1710912345678,
  "date": "2024-03-20T07:12:34.567Z",
  "programId": "bps_bench",
  "programName": "BPSベンチプレスプログラム",
  "week": 1,
  "day": 1,
  "exercises": [
    {
      "type": "bench_press",
      "name": "ベンチプレス",
      "sets": [
        { "weight": 100, "reps": 1, "completed": true }
      ]
    }
  ]
}
```

## 4. デザインガイドライン (UI/UX)

- **配色テーマ**: ダークネイビー（`#050a14`）を基調とし、アクセントカラーにゴールド（`#ffc627`）とブルー（`#004e98`）を使用します。
- **グラデーション**: カードやボタンには `linear-gradient(135deg, ...)` を多用し、プレミアム感を演出します。
- **タイポグラフィ**: `Inter` フォントを標準とし、数値（重量など）は太字（`900`）で強調します。
